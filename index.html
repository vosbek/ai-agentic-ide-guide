Core Workflows

Define Project Context
Create a comprehensive project brief including tech stack, requirements, constraints, and coding standards.
Setup AI Configuration
Configure your IDE settings, create .cursorrules or equivalent configuration files for consistent AI behavior.
Establish File Structure
Generate initial project structure with AI assistance, ensuring it follows your architectural patterns.
Create Base Templates
Generate boilerplate code, configuration files, and development setup scripts.

Effective Patterns

Context-Rich Prompts

Provide comprehensive context for better AI responses:

  • Current codebase patterns and conventions
  • Technology stack and version constraints
  • Performance and security requirements
  • Team coding standards and preferences
  • Existing architecture and design patterns

Incremental Development

Build features incrementally with AI assistance:

  • Start with minimal working implementation
  • Add features one at a time
  • Test and validate at each step
  • Refactor and optimize continuously
  • Document decisions and trade-offs

Quality Assurance

Maintain code quality with systematic checks:

  • Define acceptance criteria upfront
  • Generate comprehensive test suites
  • Implement code review checklists
  • Validate security and performance
  • Maintain documentation standards

Iterative Improvement

Continuously improve code with AI feedback:

  • Regular code analysis and suggestions
  • Performance profiling and optimization
  • Security vulnerability scanning
  • Technical debt identification
  • Refactoring opportunities assessment

Practical Examples

API Development Example

Example prompt for generating a REST API endpoint:

Create a REST API endpoint for user authentication that:

- Uses Express.js and TypeScript
- Implements JWT token-based authentication
- Includes email/password validation
- Returns proper HTTP status codes
- Includes error handling and logging
- Follows our existing API patterns in /src/routes/
- Uses our UserService class for database operations
- Includes rate limiting and security headers
- Generates OpenAPI documentation
- Includes comprehensive unit tests

Database Migration Example

Example prompt for database schema changes:

Create a database migration for adding user roles that:

- Uses Prisma with PostgreSQL
- Adds a 'roles' table with id, name, permissions columns
- Creates a many-to-many relationship with users
- Includes proper indexes for performance
- Provides rollback migration
- Seeds default roles (admin, user, moderator)
- Updates existing TypeScript types
- Includes migration tests
- Follows our existing schema patterns

Component Development Example

Example prompt for React component creation:

Create a reusable DataTable component that:

- Uses React with TypeScript
- Implements sorting, filtering, and pagination
- Supports custom column definitions
- Uses Tailwind CSS for styling
- Includes loading and error states
- Follows our component patterns in /src/components/
- Uses our custom hooks for data fetching
- Includes accessibility features (ARIA labels)
- Supports responsive design
- Includes Storybook stories and unit tests
